# /api/v1/companies : (Companies)
1) /register :                  creates company + admin user atomically; returns inviteCode  (POST)
2) / :                          Admin creates an additional company                          (POST)
3) / :                          List all active companies                                    (GET)
4) /:id :                       Get a single company by ID                                   (GET)
5) /:id :                       Update company details                                       (PATCH)
6) /:id/regenerate-invite:      Regenerate the employee invite code (invalidates old one)    (POST)

#  /api/v1/users : (Users) 
1) /register                  : Employee self-registration using inviteCode — role forced to "employee" (POST)
2) /login                     : Login with companyId + email + password — tenant-scoped                 (POST)
3) /                          : Admin manually creates manager or employee(inherits admin's companyId)  (POST)
4) /                          : List all users in the company; filter by ?role=                         (GET)
5) /:id                       : Get a single user                                                       (GET)
6) /:id                       : Update name / department / designation / isActive — role field not editable here    (PATCH)
7) /:id                       : Soft-delete user (sets isActive = false)                                (DELETE)

# /api/v1/projects : (Projects)
1) /                          : Create project + atomically creates manager's EmployeeProject record. Requires managerId (must be a registered employee) (POST)
2) /                          : Admin sees all company projects; employee with projectRole="manager" sees only their own (GET)
3) /my                        : Employee sees all projects they are assigned to (any projectRole) (GET)
4) /:id                       : Get single project — employee must be assigned to it (GET)
5) /:id                       : Update project — employee must have projectRole="manager" on this project (PATCH)
6) /:id/manager               : Re-assign a different employee as project manager (demotes old manager to member) (PATCH)

# /api/v1/projects/:id/employees : (Project Members)
1) /                          : Assign employee to project. body: { employeeId, projectRole: "manager"|"member" }. Assigning as manager auto-demotes the existing one (POST)
2) /                          : List all members on this project with their projectRole (GET)
3) /:eid/role                 : Project manager elevates a member to sub-manager (or resets back). Can only set "sub-manager"|"member" — not "manager" (PATCH)
4) /:eid                      : Remove employee from project (soft — isActive = false) (DELETE)

# /api/v1/tasks : (Tasks)
1) /                          : Create and assign a task. Caller must have projectRole "manager" or "sub-manager". assignedTo must be on the project (POST)
2) /                          : List tasks for a project — ?projectId= required. Caller must be manager or sub-manager. Filter: ?status= ?priority= (GET)
3) /kanban                    : Tasks grouped as { pending, in-progress, completed } — ?projectId= required (GET)
4) /my                        : All tasks assigned to the logged-in employee across all projects. Filter: ?status= (GET)
5) /:id                       : Get single task — manager/sub-manager or the assigned employee only (GET)
6) /:id                       : Update task metadata: title, priority, deadline, estimatedHours, tags (PATCH)
7) /:id/status                : Employee sets task status → triggers project progress recalculation (PATCH)
8) /:id/progress              : Employee logs actualHours + completionPercentage → triggers project progress recalculation (PATCH)
9) /:id                       : Delete task (DELETE)
